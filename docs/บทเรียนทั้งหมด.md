# 📚 สรุปบทเรียน React + TypeScript ทั้ง 32 บท

เรียงตามลำดับที่เรียนจริง — แต่ละบทมี "เรียนเรื่องอะไร" + "จุดสำคัญ/โค้ด"

---

## บทที่ 1 — Component & JSX
**เรียน:** Component คือฟังก์ชันที่ `return` JSX (ชื่อขึ้นต้นตัวใหญ่)
```tsx
function App() { return <h1>Get started</h1>; }
```
- JSX ใช้ `{}` แทรกค่า JS, ใช้ `className` แทน `class`, ปิดแท็กให้ครบ

## บทที่ 2 — State & `useState`
**เรียน:** State = ความจำของ component พอเปลี่ยน React วาดจอใหม่
```tsx
const [count, setCount] = useState(0);
onClick={() => setCount(count + 1)}
```
- ต้องเปลี่ยนผ่าน `setXxx` เสมอ ห้ามแก้ตัวแปรตรงๆ

## บทที่ 3 — Props
**เรียน:** ส่งข้อมูลจาก component แม่ → ลูก (พร้อมกำหนด type)
```tsx
type GreetingProps = { name: string };
function Greeting({ name }: GreetingProps) { return <p>{name}</p>; }
```
- TypeScript เตือนทันทีถ้าส่ง prop ผิด/ขาด

## บทที่ 4 — แยกไฟล์ (import / export)
**เรียน:** 1 component = 1 ไฟล์ เพื่อความเป็นระเบียบ
```tsx
export function Greeting() {...}        // ส่งออก
import { Greeting } from "./Greeting";  // นำเข้า (named export มีปีกกา)
```

## บทที่ 5 — แสดงลิสต์ด้วย `.map()`
**เรียน:** แปลง array ข้อมูล → array ของ JSX
```tsx
{friends.map((name) => <Greeting key={name} name={name} />)}
```
- ทุก `.map()` ต้องมี `key` ที่ไม่ซ้ำ

## บทที่ 6 — สร้างแอป Todo (State + form)
**เรียน:** useState กับ array, controlled input, อัปเดต array แบบ immutable
```tsx
const [todos, setTodos] = useState<string[]>([]);
setTodos([...todos, text]);   // เพิ่มแบบสร้าง array ใหม่
```

## บทที่ 7 — CSS ใน React
**เรียน:** 2 วิธี — inline style `style={{}}` กับ `className` + ไฟล์ CSS
- `style={{ padding: 20 }}` (ปีกกา 2 ชั้น, camelCase), `import "./TodoApp.css"`

## บทที่ 8 — ติ๊กงานเสร็จ (เปลี่ยน data shape)
**เรียน:** อัปเกรดข้อมูลจาก `string` → `object` เมื่อต้องเก็บข้อมูลมากขึ้น
```tsx
type Todo = { text: string; done: boolean };
todos.map((t) => t === target ? {...t, done: !t.done} : t)
```

## บทที่ 9 — localStorage + `useEffect`
**เรียน:** บันทึกข้อมูลถาวร (ไม่หายตอนรีเฟรช)
```tsx
useEffect(() => {
  localStorage.setItem("my-todos", JSON.stringify(todos));
}, [todos]);
```
- `JSON.stringify` (เก็บ) / `JSON.parse` (โหลด), lazy init ใน useState

## บทที่ 10 — แตก Component ย่อย + ส่งฟังก์ชันเป็น Props
**เรียน:** ดึง `<li>` ออกเป็น `TodoItem`, ส่ง callback ลงไป
```tsx
type Props = { onToggle: () => void; onDelete: () => void };
```
- หลักการ: ข้อมูลไหลลง (props), เหตุการณ์ไหลขึ้น (callbacks)

## บทที่ 11 — ตัวกรอง (Filter / derived state)
**เรียน:** คำนวณค่าสดจาก state ไม่เก็บซ้ำเป็น state ใหม่
```tsx
const visibleTodos = todos.filter((t) => filter === "active" ? !t.done : ...);
```
- union type: `useState<"all" | "active" | "done">("all")`

## บทที่ 12 — ใช้ `id` เฉพาะตัว (แก้ bug)
**เรียน:** แก้ปัญหา index เพี้ยนตอนกรอง → ใช้ `id` ติดตัวถาวรแทน
```tsx
{ id: Date.now(), text, done: false }
todos.filter((t) => t.id !== id)
```

## บทที่ 13 — React Router (หลายหน้า)
**เรียน:** ทำเว็บหลายหน้าโดยไม่โหลดใหม่ทั้งหน้า
```tsx
<BrowserRouter><Routes>
  <Route path="/todos" element={<TodoApp />} />
</Routes></BrowserRouter>
<Link to="/about">  // ใช้ Link ไม่ใช่ <a>
```

## บทที่ 14 — NavLink + หน้า 404
**เรียน:** ไฮไลต์เมนูหน้าปัจจุบัน + ดักทุก URL ที่ไม่ตรง
```tsx
<NavLink to="/" style={({ isActive }) => ({...})}>
<Route path="*" element={<NotFound />} />  // วางท้ายสุด
```

## บทที่ 15 — `useParams` (หน้ารายละเอียด)
**เรียน:** หน้าแบบ `/todos/:id` แสดงรายการเฉพาะตัว
```tsx
<Route path="/todos/:id" element={<TodoDetail />} />
const { id } = useParams();   // ได้ค่าเป็น string เสมอ → Number(id)
```

## บทที่ 16 — รวม type ไว้ที่เดียว (`import type`)
**เรียน:** สร้าง `types.ts` แล้ว import ใช้ทุกไฟล์ (DRY)
```tsx
export type Todo = {...};
import type { Todo } from "./types";   // type หายตอน build
```

## บทที่ 17 — Fetch API จริง
**เรียน:** ดึงข้อมูลจาก server + จัดการ 3 สถานะ (loading/error/success)
```tsx
fetch(url).then((r) => r.json()).then(setData).catch(setError)
```

## บทที่ 18 — async/await
**เรียน:** เขียน fetch ให้อ่านง่ายกว่า `.then()`
```tsx
async function load() {
  try { const res = await fetch(url); ... }
  catch (err) { if (err instanceof Error) setError(err.message); }
  finally { setLoading(false); }
}
```
- useEffect ห้ามเป็น async → สร้างฟังก์ชัน async ข้างในแล้วเรียก

## บทที่ 19 — Custom Hook (`useFetch`)
**เรียน:** แยก logic การโหลดออกมาใช้ซ้ำได้ + generic
```tsx
function useFetch<T>(url: string) { ...; return { data, loading, error }; }
```

## บทที่ 20 — PostDetail (API + useParams)
**เรียน:** รวม useFetch + useParams → โหลดข้อมูลตัวเดียวจาก API
```tsx
useFetch<Post>(`.../posts/${id}`)   // url เปลี่ยน → useFetch โหลดใหม่เอง
```

## บทที่ 21 — Context API
**เรียน:** แชร์ state ทั้งแอป แก้ปัญหา prop drilling
```tsx
createContext → <Provider value={{...}}> → useContext (ผ่าน custom hook useTodos)
```
- ย้าย todos + logic ทั้งหมดไปไว้ใน `TodoProvider`

## บทที่ 22 — แก้ไขงาน (Edit Mode)
**เรียน:** สลับ "โหมดดู" ↔ "โหมดแก้" ด้วย state `isEditing`
```tsx
if (isEditing) return (<input .../> + ปุ่มบันทึก/ยกเลิก);
return (โหมดปกติ);   // conditional rendering แบบ return 2 ทาง
```

## บทที่ 23 — Empty State
**เรียน:** แสดงข้อความเมื่อลิสต์ว่าง (UX ที่ดี)
```tsx
{visible.length === 0 ? <p>ยังไม่มีงาน</p> : <ul>...</ul>}
```

## บทที่ 24 — CSS ร่วม (Shared styles)
**เรียน:** สร้าง `pages.css` class กลาง (`.page`) ใช้หลายหน้า
- import ที่ main.tsx ครั้งเดียว = global ทั้งแอป

## บทที่ 25 — `useRef`
**เรียน:** เข้าถึง DOM ตรงๆ (เช่น โฟกัส input อัตโนมัติ)
```tsx
const inputRef = useRef<HTMLInputElement>(null);
useEffect(() => { inputRef.current?.focus(); }, []);
<input ref={inputRef} />
```
- เปลี่ยน ref ไม่ทำให้ re-render (ต่างจาก useState)

## บทที่ 26 — `useMemo` & `useCallback`
**เรียน:** จำผลลัพธ์ไว้กันคำนวณซ้ำ (performance)
```tsx
const visible = useMemo(() => todos.filter(...), [todos, filter]);
const handleAdd = useCallback(() => {...}, [text]);
```
- ⚠️ อย่าใช้พร่ำเพรื่อ — ใช้เมื่อมีปัญหา performance จริง

## บทที่ 27 — Tailwind CSS
**เรียน:** ใส่ style ด้วย utility class ใน JSX
```tsx
<div className="max-w-md mx-auto p-6 bg-gray-900 rounded-xl">
```
- `hover:`, `focus:` prefix, dynamic class ด้วย ternary

## บทที่ 28 — React Hook Form + Zod
**เรียน:** ทำฟอร์มจริงจัง + validate
```tsx
const schema = z.object({ name: z.string().min(2, "...") });
useForm<FormInput, unknown, FormData>({ resolver: zodResolver(schema) });
<input {...register("name")} />
```
- `coerce` → input type ≠ output type → ใช้ `z.input` / `z.output`

## บทที่ 29 — TanStack Query (`useQuery`)
**เรียน:** จัดการ API ระดับโปร — มี cache อัตโนมัติ
```tsx
const { data, isLoading, error } = useQuery({
  queryKey: ["posts"], queryFn: async () => (await fetch(url)).json(),
});
```
- ต้องครอบด้วย `<QueryClientProvider>`

## บทที่ 30 — TanStack Query Mutations
**เรียน:** เขียน/แก้/ลบ ข้อมูล (POST/PUT/DELETE)
```tsx
const m = useMutation({
  mutationFn: async (x) => fetch(url, { method: "POST", body: JSON.stringify(x) }),
  onSuccess: () => queryClient.invalidateQueries({ queryKey: ["posts"] }),
});
m.mutate(value);
```
- หลังเขียนเสร็จ → `invalidateQueries` → รายการรีเฟรชเอง

## บทที่ 31 — การ Debug
**เรียน:** เครื่องมือหาบั๊ก
1. อ่าน error ใน Console ก่อน
2. `console.log` / `console.table`
3. React DevTools (ดู state real-time)
4. Network tab (status code + response)
5. `debugger;` (หยุดทีละบรรทัด)

## บทที่ 32 — Deploy ขึ้นเว็บจริง
**เรียน:** เอาเว็บขึ้น internet
```bash
npm run build    # → โฟลเดอร์ dist/
```
- Netlify Drop (ลาก dist) หรือ Vercel (เชื่อม GitHub → auto-deploy)
- SPA routing: `_redirects` (Netlify) / `vercel.json` (Vercel) กัน refresh 404
